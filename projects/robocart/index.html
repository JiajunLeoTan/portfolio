Overview
Project Summary

RoboCart is an autonomous wheeled robot designed for a class-wide competition. The robot navigates using ultrasonic distance sensors and HTC Vive positioning sensors, follows walls autonomously, and deploys a servo-driven attack arm triggered via I2C whisker switch input — all coordinated by an ESP32-S2 microcontroller.

The robot operates in two modes: a web-based manual control interface served directly from the ESP32 over SoftAP Wi-Fi, and a fully autonomous mode combining wall-following logic with attack sequencing. Every subsystem — firmware, control logic, electronics, and chassis — was designed and built by a three-person team.

Engineering Challenge
Debugging the Ultrasonic Failures
Root Cause Analysis
Four sensors destroyed — and why it kept happening

During development, we damaged four SEN0001 ultrasonic sensors in sequence. The motors, Vive sensors, ESP32-S2, servo motors, and buttons all remained stable — the failures were isolated to the ultrasonic sensors, which are known to be sensitive to power conditions.

After investigating the HC-SR04 datasheet, we identified the root cause: the datasheet explicitly states that no pins (Vcc, Trig, or Echo) should be connected to a power source unless the GND terminal is already connected. During testing of the wall-following code, this requirement had not been consistently followed, causing the sensors to overheat and become permanently damaged.

The fix was to replace the SEN0001 units with HC-SR04 sensors and enforce strict GND-first grounding practices across all connections. After the change, no further sensor failures occurred for the remainder of the project — confirming the hypothesis and reinforcing the importance of reading datasheets before touching hardware.
