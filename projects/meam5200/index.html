// 01 · overview
The Challenge

Two robots compete simultaneously. Each team must pick 50 mm AprilTag-labeled cubes from their own static platform and from a shared rotating turntable, then stack them on their goal platform. Scoring is Points = Value × Altitude — a perfectly aligned stack of 8 blocks (4 dynamic on top of 4 static) would yield a theoretical 28,000 points.

The core engineering challenge: dynamic blocks move continuously at 0.4 RPM. The robot must predict where a block will be by the time the arm arrives — not where it is when the camera sees it. Our strategy was to guarantee the higher-value dynamic blocks first, then combine the two stacks into a tower of 8.

// 02 · version history
How the Code Evolved

The codebase went through 7 distinct stages — from a bare ROS scaffold to a modular competition build. The naming convention tells the story: original → combined → new_check → V3 → V4. Each version solved a real problem that the previous one couldn't handle.

original_final.py 76 lines scaffold
Bare competition template — ROS node init, ArmController, ObjectDetector, move to start position, block detection stub. No logic, no functions. The blank canvas.
ROS ArmController ObjectDetector
original_final_combined.py 824 lines first implementation
First real implementation. Core logic modularized into functions: detect_static_block_positions, stack_static_blocks, stack_dynamic_blocks, combine_stack. Static block pipeline working end-to-end; dynamic blocks handled with a skip_turntable flag as a placeholder for prediction.
static pipeline dynamic pipeline modular functions combine_stack
final.py  /  final_Jason.py  /  finalEric.py ~640–1067 lines each personal branches
Parallel development. Each teammate worked on their own branch. IK_position_null_Leo.py and IK_position_null_Fast.py reflect Leo's IK optimizations. Jason's branch explored dynamic pickup; Eric's branch refined gripper logic. These were never intended to be deployed — they're the experimentation layer.
IK_position_null_Leo IK_position_null_Fast FK/IK pipeline gripper logic
final_combined.py 1,516 lines first merge
All branches merged. stack_dynamic_blocks now takes detector as an explicit parameter instead of the skip_turntable flag. check_grab_succeed added as a new failsafe. Gripper close position tuned from 0.04 m to 0.045 m. Team calibration offsets broken out separately for red and blue.
check_grab_succeed calibration offsets gripper tuning detector injection
final_combined_new_check.py 1,561 lines camera failsafe
Gripper state replaced with vision-based verification. After each block placement, the arm moves to a position where it can see the top of the stack. If the expected block isn't visible in two consecutive camera frames, it's assumed the pickup failed and the robot attempts that height again. Gripper width feedback was too noisy in hardware; the camera was more reliable.
camera re-detection redetect_dynamic_blocks vision failsafe
final_combined_new_check_V3.py 1,619 lines turntable prediction
The key breakthrough. rotate_block_on_turntable now actually computes a predicted_pose: the block's future position when the arm arrives, not where it was when the camera saw it. Uses ttotal = tmovement + 5 s calibration offset and a Rz(θ) rotation about the turntable center. A collision safety threshold was also added — the robot skips pickup if the block's x-axis orientation falls outside a safe window to prevent the gripper impacting the block's side.
predict future pose Rz(θ) rotation calculate_time_to_reach collision threshold t_calibration = 5s
final_combined_new_check_V4.py 1,066 lines competition build
Stripped for competition. combine_stack removed — the timed environment didn't reliably allow the full 8-block procedure. Cleaner execution path, same prediction core. Back to ~1,066 lines but with all the hardware-tested refinements baked in.
competition build combine_stack removed leaner execution

competition post-mortem
We entered the playoffs as top seed. Late-round performance degraded for two reasons: (1) camera calibration offsets valid the previous day drifted on competition day due to lighting changes, and (2) an opposing team discovered a sweeping strategy that pushed dynamic blocks to the center of the turntable — outside our fixed viewing cone. A timeout fallback to static-only pickup would have prevented the time loss in the final matches.
